unqueue is a real-time monitoring dashboard for [BullMQ](https://docs.bullmq.io) — the Node.js queue library built on Redis. It connects directly to your Redis instance and surfaces queue state, job details, failure reasons, stack traces, and worker logs without requiring any agents, code changes, or instrumentation.

## How it works

unqueue reads BullMQ's Redis key schema directly. When you paste a Redis connection URL, it scans for all BullMQ queues using the key prefix you configure, then subscribes to BullMQ's `QueueEvents` channel to receive live updates as jobs move through the queue.

Your job data never leaves your Redis instance. Every request reads directly from Redis on demand — nothing is persisted or cached on unqueue's servers.

## What you can do

- **Inspect jobs** — view input data, return values, stack traces, and worker logs for any job in any state
- **Monitor queues** — see active, waiting, delayed, and failed counts update in real time
- **Take action** — retry failed jobs, pause queues, drain backlogs, and clean completed jobs from the dashboard
- **Get alerted** — configure Discord webhooks to be notified when failure rates spike or queues stall

## Requirements

- BullMQ v3 or v4
- Redis 6+ (including Redis Cloud, Upstash, ElastiCache, Aiven, Azure Cache, and self-hosted)
- Node.js workers using standard BullMQ queue names and key prefixes
